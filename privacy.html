Privacy Policy

Effective date: February 2026  ·  Laneway Bee Chrome Extension v1.0.0

Laneway Bee is an internal tool built for organisations to record and analyse their own Google Meet sessions. It is not a public product — access is restricted to employees of the organisation whose administrator has configured the extension with a private API key. This policy explains exactly what data is collected, how it is used, and how it is protected.

1. Who This Policy Applies To

This policy applies to employees and team members of organisations that have deployed Laneway Bee internally. The extension is only functional after an administrator supplies a valid API key obtained from the organisation's Laneway dashboard. It is not intended for use by the general public.

All Google Meet sessions recorded through this extension involve only employees who have been sent a calendar invite directly from the organisation's Laneway dashboard. Meeting links are not shared outside the organisation.

2. Data We Collect

The extension collects the following categories of data only while a Google Meet session is active and the user has started a recording:

Data Source Purpose
Audio & video recording Browser screen/tab capture (user-initiated) Meeting replay, transcription, analysis
Meeting ID & title Google Meet page URL and DOM Linking the recording to the correct calendar event
Participant names Google Meet participant panel DOM Engagement analytics per attendee
Join & leave times Participant panel DOM Attendance tracking
Microphone state (muted / active) Participant panel DOM Approximate speaking-time analytics
Camera on/off duration Participant panel DOM Engagement analytics
API key & backend URL Entered by user in extension settings Authenticating with the organisation's backend
Extension settings Entered by user in extension popup Remembering recording quality preference, auto-start preference

Important The extension does not collect data when a meeting is not active. It does not read browser history, access other tabs, read emails, access contacts, or collect any data outside of an active Google Meet session.

3. How Data Is Used

  • Recordings are stored in the organisation's private Cloudflare R2 bucket and used solely for internal review, transcription, and analysis.
  • Participant analytics (names, join/leave times, mic state, camera duration) are stored in the organisation's backend and surfaced only to authorised dashboard users within the same organisation.
  • Meeting metadata (meeting ID, title, recording URL, duration) is linked to the corresponding calendar event in the Laneway dashboard.
  • Settings and API key are stored locally in Chrome's sync storage and never transmitted to any third party.

No data is sold, licensed, or shared with advertisers or any external parties.

4. Where Data Is Stored

  • Recordings & participant JSON files — Cloudflare R2 object storage, accessible only via the organisation's authenticated Cloudflare Worker.
  • Analytics snapshots — Cloudflare D1 database, scoped to the organisation's Worker deployment.
  • Meeting records & participant analytics — Organisation's backend server (Render). Access is restricted by API key authentication.
  • API key & preferences — Chrome's storage.sync API, stored locally in the user's browser profile.

5. Recording Consent

Recording only begins when the user actively clicks Start Recording in the extension popup. The browser's native screen-share permission prompt is shown before any capture begins — the user must explicitly grant permission.

All participants in a recorded meeting are employees of the same organisation who received a calendar invite from the organisation's Laneway dashboard. Recording of internal company meetings is conducted in accordance with the organisation's employment policies and applicable local laws. It is the responsibility of the organisation's administrator to ensure participants have been informed that meetings may be recorded.

6. Data Retention

The organisation's administrator controls data retention. There is no automatic deletion enforced by Laneway Bee itself. Administrators can delete recordings and analytics data directly from their Cloudflare R2 bucket and backend database at any time.

7. Data Security

  • All data in transit is encrypted via HTTPS/TLS.
  • The Cloudflare Worker validates a secret API key on every request — no data is publicly accessible.
  • The Laneway backend requires a per-user API key for all extension-facing endpoints.
  • No raw API keys are logged by the backend; only SHA-256 hashes are stored for verification.

8. Third-Party Services

Laneway Bee communicates exclusively with the following services operated by or on behalf of the organisation:

  • Cloudflare — R2 object storage and D1 database (data processed under Cloudflare's Data Processing Agreement).
  • Render — Backend hosting (data processed under Render's Data Processing Agreement).
  • Google Meet — The extension reads DOM data from the Google Meet page only. It does not call any Google APIs directly; it does not access Gmail, Calendar, Drive, or any other Google service.

9. Permissions Explanation

Permission Why It Is Needed
storage Saves the API key, backend URL, and recording preferences locally in Chrome.
tabs Detects whether the currently active tab is a Google Meet page so the extension can show relevant controls in the popup.
activeTab Grants temporary access to the current tab when the user clicks the extension icon.
meet.google.com The content script is injected only into Google Meet pages to detect meetings and enable recording.
Cloudflare Worker URL Uploads recordings and participant data to the organisation's private R2 storage.
Backend URL Sends recording metadata and analytics to the organisation's Laneway backend.

10. Your Rights

As this extension is deployed within your organisation, requests relating to your personal data (access, correction, deletion) should be directed to your organisation's administrator. The administrator has full access to delete your data from the Cloudflare R2 bucket and the Laneway backend database.

11. Changes to This Policy

If this policy is updated, the effective date at the top of this page will change. Significant changes will be communicated to administrators via the Laneway dashboard.

12. Contact

For any questions about this privacy policy or how your data is handled, contact your organisation's Laneway administrator or reach out at privacy@laneway.in.
